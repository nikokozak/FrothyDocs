Tutorials are for doing one thing end to end. The guide explains the mental
model; the reference gives exact behavior. These pages start with a visible
goal and stay close to the board.

Start with [Blink an LED](/tutorials/blink-an-led/) if you are new to the
hardware path. Use [Interactive Workflow](/tutorials/interactive-workflow/) when
you want the edit-loop shape. Use [Read Inputs](/tutorials/read-inputs/) or
[Build a Small Game](/tutorials/build-a-small-game/) when you have a Froth
Machine nearby.
